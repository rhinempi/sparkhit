Dependencies

Sparkhit is written in Java and built on top of the Apache Spark. To use Sparkhit, make sure you have a Java-1.7 or higher version installed in your operating system, as well as Spark framework. Since Sparkhit runs on a distributed computer cluster, Spark cluster should also be properly configured when running Sparkhit on cluster mode. If you want to parallelize your own tools with Sparkhit, make sure they are available on all computer nodes (eg. copy your script to each node or place it in a shared file system). See use your own tool

Spark installation

There are different ways to install Spark on your computer. Please go to Spark`s Download page and choose a download method of your preference. If you would like to build Spark from source, please go to Building Spark page.

A simple way is to download Spark pre-build package (version 2.0.0) using command:
$ wget http://archive.apache.org/dist/spark/spark-2.0.0/spark-2.0.0-bin-hadoop2.6.tgz

Once download, untar the package:
$ tar zxvf spark-2.0.0-bin-hadoop2.6.tgz

Go to unpacked Spark directory. Under bin folder, you should see an executable file spark-submit:
$ cd ./spark-2.0.0-bin-hadoop2.6
$ ./bin/spark-submit

Set environment variable (ENV) SPARK_HOME to your unpacked Spark directory:
$ pwd  Then, copy listed full path.
$ vi /home/ec2-user/.bash_profile or $ vi /home/ec2-user/.profile
Here, ec2-user refers to your user name. Paste your Spark full path in the file and set as ENV SPARK_HOME 

# bash_profile
export SPARK_HOME="/vol/ec2-user/spark-2.0.0-bin-hadoop2.6"

Reload changes in the file:
$ source /home/ec2-user/.bash_profile

Now Sparkhit should be able to recognize Spark.

Sparkhit installation

Download Sparkhit package from github zipball:
$ wget https://github.com/rhinempi/sparkhit/archive/latest.zip -O ./sparkhit.zip
If you do not have wget installed:
$ curl -fsSL https://github.com/rhinempi/sparkhit/archive/latest.zip -o ./sparkhit.zip

Or tarball:
$ wget https://github.com/rhinempi/sparkhit/archive/latest.tar.gz -O ./sparkhit.tar.gz
Alternatively:
$ curl -fsSL https://github.com/rhinempi/sparkhit/archive/latest.tar.gz -o ./sparkhit.tar.gz
Or press the download buttom on top or click Download.

Once download, unzip or untar the package:
$ unzip ./sparkhit.zip
$ tar zxvf ./sparkhit.tar.gz

Go to Sparkhit directory, you should see an executable file sparkhit in ./bin folder, set its permission to executable if it is not:
$ cd ./sparkhit-latest
$ chmod 755 ./bin/sparkhit

Now, you should be able to run Sparkhit:
$ ./bin/sparkhit

./sbin
☕  Notes
  1. The executable file sparkhit is a shell script wrapped spark-sumbit with Sparkhit`s jar file. The example of full command to submit a Sparkhit application can be found in ./sbin folder.
  2. The input parameters for Sparkhit consist of options for both Spark framework and correspond Sparkhit applications. Spark options are started with two dashes -- - to configure the Spark cluster, whereas Sparkhit options are started with one dash- - to specify parameters for each Sparkhit application.
